CZDateFormatterCache

Anyone who has profiled scroll performance of a UITableView whose cells render dates knows how slow it is create NSDateFormatter instances. CZDateFormatterCache creates a global cache of NSDateFormatter instances -- created the first time you use the cache -- that can be shared by all of your UI code, as in:

NSDate *date = ...;

cell.detailTextLabel.text = [[CZDateFormatterCache mainQueueCache] localizedStringFromDate:date dateStyle:kCFDateFormatterShortStyle timeStyle:kCFDateFormatterShortStyle];

Since NSDateFormatter instances are not thread safe, you should only use the cache from your main dispatch queue. In fact, CZDateFormatterCache asserts if you try to use the cache from a different dispatch queue.
